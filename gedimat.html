Gedima'Gination




Contexte

Projet fictif : La société NEGOMAT vient tout juste d’entrer dans le réseau d’adhérents de GEDIMAT. Pour fêter cet événement, NEGOMAT a décidé d’organiser un jeu concours nommé « GEDIMA’GINATION ».

Cahier des charges : créer une application web pour participer au concours et consulter les résultats, et une application mobile pour voter pour ses participation préférées.




Technologies et outils

StarUML (logiciel de modélisation UML), HeidiSQL (outil d'administration de base de données), SQL, PHP, Java, CSS et PHPStorm (IDE).




Démarche

Le projet a été réalisé en mode agile (les missions sont regroupées en sprint).
Les premières étapes du projet (réflexion, conception UML et création de la base de données) ont été réalisé en équipe.

Le projet se découpe en 2 parties : l'application web et l'application mobile (avec l'API qui permet de communiquer)




Application Web

L'application web permet aux visiteurs de se connecter ou de créer un compte, et si la période des votes est terminée, de consulter le classement des meilleures participations quand il a été publié.

Si l'utilisateur est connecté en tant que participant, il peut poster une réalisation quand les conditions sont remplies (il n'a pas déjà participé et il est dans la période de participation)

Si l'utilisateur est connecté en tant que gestionnaire, il générer et publier le classement des participations quand les conditions sont remplies (la période des votes est terminée)




Formulaire de participation

Création de la maquette du formulaire et formulaire de participation final.




Inscription/connexion

Possibilité pour un visteur de créer un compte, et connexion au site (participant ou gestionnaire)




Générer/publier

Le gestionnaire peut générer un classement des participations selon le nombre de votes, puis publier le résultat pour les visiteurs (ne sont visible que les 3 meilleurs)





Application Mobile

Les votes se font en magasin via une tablette.

Les clients utilisent le code fournit sur leur ticket de caisse pour s'identifier, ils peuvent ainsi attribuer un nombre de "Gaimes" à leur 3 participations préférées (les photos et les identifiants défilent sur un écran devant la tablette)





Importation/exportation

Le gestionnaire peut importer les réalisation enregistrées via le site web, et exporter les données pour le résultat des votes




Vote

Un client du magasin doit renseigner des informations avant de voter, il vote pour ses 3 réalisations préférées








API

L'API permet à l'application web et l'application mobile de communiquer.




Bilan

Ce projet a été long à réaliser mais il m'a permis de travailler sur de nombreux aspects techniques et j'ai pu faire communiquer des applications sur des supports différents via l'API.

Il a été très enrichissant et stimulant malgré les difficultées rencontrées en Java.
